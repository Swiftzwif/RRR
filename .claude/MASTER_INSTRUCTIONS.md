## Decision Framework

When making any decision, ask:

1. **Does this serve transformation?**
   - If no → Don't build it
   - If yes → How can we maximize impact?

2. **Is this Apple-grade quality?**
   - If no → Polish until it is
   - If yes → Can we make it even better?

3. **Will this scale to 10,000+ users?**
   - If no → Redesign architecture
   - If yes → Future-proof it further

4. **Is this the simplest solution?**
   - If no → Simplify ruthlessly
   - If yes → Document clearly
